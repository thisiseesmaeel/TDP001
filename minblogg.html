Lösa en konflikt

ctrl+d för att gå tillbaka till mitt eget system.

emacs README.md
för att öppna med emacs. Ändrade README.md filen, sedan pullar man med följande: 
git pull

En merge-conflict inträffar när Git inte förstår skillnaderna mellan två commit.
När jag fick en merge-conflict så anledningen till det var att README.md filen på servern och i det lokala var inte lika. Det var för att jag hade redan ändrat README.md filen på servern innan och hade pushat den till Git, och nu när jag ändrar README.md filen i det lokala systemet och pullar från Git så hände det merge-conflict. För att fixa merge-conflictet så öppnade jag filen med text editor och tog bort det onödiga texter som var highlightad och sparade filen.
